# `/bot` Skill Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a Claude Code slash-command skill (`/bot`) that lists installed Discord bots and runs start/stop/restart/status against any one of them.

**Architecture:** A bash helper script (`bin/bot`) does all the work — arg parsing, launchctl invocations, status probes. A thin SKILL.md tells Claude Code to invoke that script when the user types `/bot`. The script is also usable directly from the terminal (`bot dsbot restart`).

**Tech Stack:** bash (script + tests), launchd (start/stop/restart), Discord REST API (status probe), standard Unix tools (grep, sed, lsof, curl, ps).

**Spec:** `docs/superpowers/specs/2026-07-01-bot-skill-design.md`

## Global Constraints

- Skill location: `~/.claude/skills/bot/`
- Files:
  - `~/.claude/skills/bot/SKILL.md`
  - `~/.claude/skills/bot/bin/bot`
  - `~/.claude/skills/bot/tests/test_bot.sh`
- Discovery: scan `~/Library/LaunchAgents/com.user.*-claude.plist`
- Action mappings: `start` → `launchctl load`; `stop` → `launchctl unload`; `restart` → `launchctl kickstart -k gui/501/com.user.<bot>-claude`; `status` → composite (launchd PID + claude PID + bun PID + gateway TCP + API ping)
- Exit codes: `0` = success, `2` = invalid action, `3` = bot not installed, anything else = passthrough from launchctl
- Output: stdout = data, stderr = errors/warnings. Token from `.env` MUST NOT be echoed.
- `bin/bot` must work both as `~/.claude/skills/bot/bin/bot <args>` and via PATH (`bot <args>`)

# Tests for ~/.claude/skills/bot/bin/bot
set -euo pipefail

PASS=0; FAIL=0

assert_eq() {
    local desc="$1" expected="$2" actual="$3"
    if [ "$expected" = "$actual" ]; then
        echo "PASS: $desc"; PASS=$((PASS+1))
    else
        echo "FAIL: $desc"; echo "  expected: $expected"; echo "  actual:   $actual"; FAIL=$((FAIL+1))
    fi
}

assert_contains() {
    local desc="$1" needle="$2" haystack="$3"
    if echo "$haystack" | grep -qF -- "$needle"; then
        echo "PASS: $desc"; PASS=$((PASS+1))
    else
        echo "FAIL: $desc (looking for '$needle')"; echo "  got: $haystack"; FAIL=$((FAIL+1))
    fi
}

out=$("$BOT_BIN" list 2>&1) && ec=$? || ec=$?

- [ ] **Step 4: Write `bin/bot` with `list` + argument parsing**

Create `~/.claude/skills/bot/bin/bot`:

```bash
#!/usr/bin/env bash
# ~/.claude/skills/bot/bin/bot — manage Discord bots installed via launchd
#
# Usage:
#   bot                  list installed bots
#   bot <name>           status of one bot (default action when one arg given)
#   bot <name> <action>  action = start | stop | restart | status
set -euo pipefail

PLIST_DIR="$HOME/Library/LaunchAgents"
PLIST_GLOB="$PLIST_DIR/com.user.*-claude.plist"

# Strip the plist filename down to the bot name.
# com.user.plannerbot-claude.plist → plannerbot
bot_name_from_plist() {
    basename "$1" | sed -e 's|^com\.user\.||' -e 's|-claude\.plist$||'
}

# Print bot names, one per line, sorted unique.
list_bots() {
    local p
    for p in $PLIST_GLOB; do
        [ -f "$p" ] || continue
        bot_name_from_plist "$p"
    done | sort -u
}

# Print the plist path for a named bot (if it exists).
# Returns 0 if found, 1 if not.
plist_path_for() {
    local name="$1" p
    for p in $PLIST_GLOB; do
        [ -f "$p" ] || continue
        if [ "$(bot_name_from_plist "$p")" = "$name" ]; then
            echo "$p"
            return 0
        fi
    done
    return 1
}

# --- arg parsing ---
ACTION="${1-}"
NAME="${2-}"

# No args → list
if [ $# -eq 0 ]; then
    list_bots
    exit 0
fi

# "list" keyword → list
if [ "$ACTION" = "list" ]; then
    list_bots
    exit 0
fi

# One arg → bot name with default action "status"
if [ $# -eq 1 ]; then
    NAME="$ACTION"
    ACTION="status"
fi

# Validate action
case "$ACTION" in
    start|stop|restart|status) ;;
    *) echo "ERROR: unknown action '$ACTION' (use: start|stop|restart|status)" >&2
       exit 2 ;;
esac

# Validate bot exists
PLIST_PATH="$(plist_path_for "$NAME" || true)"
if [ -z "$PLIST_PATH" ]; then
    echo "ERROR: bot '$NAME' not installed (no plist at $PLIST_DIR/com.user.$NAME-claude.plist)" >&2
    exit 3
fi

# Dispatch (Tasks 2/3 add start/stop/restart/status implementations here)
echo "ERROR: action '$ACTION' not yet implemented" >&2
exit 4
```

In `~/.claude/skills/bot/bin/bot`, find the final block (after `PLIST_PATH=...`):

```bash
# --- action dispatch ---
case "$ACTION" in
    start)
        # Idempotent: if already loaded, say so.
        if launchctl list | grep -q "com.user.$NAME-claude"; then
            pid=$(launchctl list | awk -v label="com.user.$NAME-claude" '$3==label {print $1}')
            echo "INFO: $NAME already running (PID $pid)"
            exit 0
        fi
        if launchctl load "$PLIST_PATH"; then
            echo "✓ $NAME started"
        else
            echo "ERROR: launchctl load failed for $NAME" >&2
            exit 1
        fi
        ;;
    stop)
        if ! launchctl list | grep -q "com.user.$NAME-claude"; then
            echo "INFO: $NAME already stopped"
            exit 0
        fi
        if launchctl unload "$PLIST_PATH"; then
            echo "✓ $NAME stopped"
        else
            echo "ERROR: launchctl unload failed for $NAME" >&2
            exit 1
        fi
        ;;
    restart)
        # kickstart -k works whether the service is loaded or not; restarts if loaded.
        if launchctl kickstart -k "gui/501/com.user.$NAME-claude" 2>/dev/null; then
            echo "✓ $NAME restarted"
        else
            # kickstart fails when not loaded; fall back to load.
            if launchctl load "$PLIST_PATH" 2>/dev/null; then
                echo "✓ $NAME started (was not running)"
            else
                echo "ERROR: kickstart and load both failed for $NAME" >&2
                exit 1
            fi
        fi
        ;;
    status)
        # Implemented in Task 3
        echo "ERROR: status not yet implemented" >&2
        exit 4
        ;;
esac
```

```bash
    status)
        echo "$NAME"
        # launchd line
        if launchctl_line=$(launchctl list | awk -v label="com.user.$NAME-claude" '$3==label {print $1, $2}'); then
            if [ -n "$launchctl_line" ]; then
                echo "  launchd:   $launchctl_line"
            else
                echo "  launchd:   not loaded"
            fi
        else
            echo "  launchd:   not loaded"
        fi

        # claude line — find a claude process whose argv contains the bot name
        claude_line=$(ps aux | grep -E '[c]laude.*--channels' | grep -- "-effort " | grep "discord-$NAME" | head -1)
        if [ -n "$claude_line" ]; then
            claude_pid=$(echo "$claude_line" | awk '{print $2}')
            claude_effort=$(echo "$claude_line" | grep -oE -- '--effort [a-z]+' | awk '{print $2}')
            echo "  claude:    $claude_pid  --effort $claude_effort"
        else
            echo "  claude:    <not running>"
        fi

        # bun line — find the bun process for this bot (its parent is the matching claude PID)
        if [ -n "$claude_pid" ]; then
            bun_line=$(ps aux | awk -v ppid="$claude_pid" '$2==ppid && /bun.*server\.ts/ {print $2, $11, $12, $13}')
            if [ -n "$bun_line" ]; then
                echo "  bun:       $bun_line"
            else
                echo "  bun:       <not running>"
            fi
        else
            echo "  bun:       <not running>"
        fi

        # gateway line — count ESTABLISHED TCP connections from any bun process (best-effort)
        bun_pids=$(ps aux | awk '/bun.*server\.ts/ && !/awk/ {print $2}')
        gw_count=0
        for bp in $bun_pids; do
            n=$(lsof -nP -p "$bp" 2>/dev/null | grep -c 'TCP.*ESTAB.*->.*:443' || true)
            gw_count=$((gw_count + n))
        done
        if [ "$gw_count" -gt 0 ]; then
            echo "  gateway:   $gw_count ESTABLISHED TCP to Discord"
        else
            echo "  gateway:   <not connected>"
        fi

        # api line — token-based ping; never echo the token
        ENV_FILE="$HOME/.claude/channels/discord-$NAME/.env"
        if [ -r "$ENV_FILE" ]; then
            TK=$(grep -E '^DISCORD_BOT_TOKEN=' "$ENV_FILE" | cut -d= -f2-)
            if [ -n "$TK" ]; then
                api_resp=$(curl -sS -o /dev/null -w 'HTTP %{http_code} in %{time_total}s' \
                    -H "Authorization: Bot $TK" \
                    https://discord.com/api/v10/users/@me 2>&1) || api_resp="<curl failed>"
                echo "  api:       $api_resp"
            else
                echo "  api:       <token unreadable>"
            fi
        else
            echo "  api:       <token unreadable>"
        fi
        ;;
```

# /bot — Discord bot management

Invokes `~/.claude/skills/bot/bin/bot` to manage bots installed via `claude-code-discord-bot-setup`.

## Usage

- `/bot` — list installed bots
- `/bot <name>` — status of one bot (PID, gateway, API)
- `/bot <name> start|stop|restart|status` — manage

## Behavior

1. Parse `<name>` and `<action>` from the user's request
2. Run `~/.claude/skills/bot/bin/bot <args>` with the parsed args
3. Display the script's stdout verbatim
4. If the script exits non-zero, surface its stderr and exit code

## Notes

- Bots are discovered by scanning `~/Library/LaunchAgents/com.user.*-claude.plist`. New bots installed via `install.sh` appear automatically.
- `start` / `stop` map to `launchctl load` / `unload`. `restart` maps to `launchctl kickstart -k` (single command, works whether loaded or not).
- `status` checks five independent signals of bot health: launchd PID, claude PID, bun PID, Discord gateway TCP, Discord API ping.
